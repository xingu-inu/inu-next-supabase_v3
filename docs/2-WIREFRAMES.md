**Step 2: WHAT 구체화 (고정 옵션 + AI 추천)**

```text
┌─────────────────────────────────────────────────────────────┐
│  🦭 INU     [📋 카드] [🌳 트리] [📅 캘린더]      [📝] 👤    │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌──────────┐                                               │
│  │ 💪 운동  │  ← 카테고리 노드 (생성됨)                     │
│  └────┬─────┘                                               │
│       │                                                     │
│       ▼  🦭 "어떤 운동이요?"                                │
│       │  (✅ 복수 선택 가능 · 각각 독립 Goal)                │
│       │                                                     │
│       │  ── 자주 선택되는 옵션 ────────────────────         │
│       │  ┌───────┐ ┌───────┐ ┌───────┐ ┌───────┐           │
│       │  │  🏃   │ │  🏋️   │ │  🏊   │ │  🧘   │           │
│       │  │ 러닝  │ │ 헬스  │ │ 수영  │ │ 요가  │           │
│       │  └───────┘ └───────┘ └───────┘ └───────┘           │
│       │                                                     │
│       │  ── 🤖 AI 추천 ──────────────────────────          │
│       │  ┌───────────┐ ┌───────────┐ ┌───────────┐         │
│       │  │  🎾       │ │  🚴       │ │  ⛰️        │         │
│       │  │ 테니스    │ │ 자전거    │ │ 등산       │         │
│       │  │ ·········  │ │ ·········  │ │ ·········  │         │
│       │  └───────────┘ └───────────┘ └───────────┘         │
│       │  (사용자 맥락 기반 개인화 추천)                       │
│       │                                                     │
│       │  ┌─────────────────┐                                │
│       │  │  + 직접 입력     │                                │
│       │  └─────────────────┘                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘

── 옵션 구분 ──────────────────────────────────────────────────
* 고정 옵션: 카테고리별 사전 정의된 인기 항목 (항상 표시)
* AI 추천: 사용자 기존 Goal/WHY 맥락 기반 개인화 (점선 테두리)
* AI 로딩 중: 고정 옵션 먼저 표시, AI 추천 영역 skeleton shimmer
* AI 실패 시: 고정 옵션만 표시 + "AI 추천을 불러오지 못했어요" 안내
* 복수 선택 시 선택된 항목에 체크 표시, 각각 독립 Goal 생성
각 선택마다 노드 생성 → 아래로 분기 → 다음 옵션 표시
뒤로가기: 생성된 노드를 클릭하면 해당 선택을 재변경 가능
```

| 💕 관계 | 가족, 연인, 친구, 네트워킹 |
| 🎨 취미 | 그림, 음악, 요리, 사진 |
| 🧘 습관 | 기상, 명상, 식습관, 수면 |
| 📖 독서 | 자기계발, 소설, 전공, 에세이 |
| ✨ 기타 | (고정 옵션 없음, AI 추천 + 직접 입력만) |

```text
┌─────────────────────────────────────────────────────────────┐
│  🦭 INU     [📋 카드] [🌳 트리] [📅 캘린더]      [📝] 👤    │

│       │                                                     │
│  ┌────▼─────┐                                               │
│  │  러닝    │  ← WHAT 구체화 노드 (생성됨)                  │
│  └────┬─────┘                                               │
│       │                                                     │
│       ▼  🦭 "왜 러닝을 하고 싶으세요?"                      │
│       │  (✅ 복수 선택 가능)                                 │
│       │                                                     │
│       │  ── 🤖 AI 추천 WHY ──────────────────────          │
│       │  ┌───────┐ ┌───────────┐ ┌───────────┐             │
│       │  │  ❤️   │ │    🎯     │ │    😌     │             │
│       │  │ 건강  │ │ 다이어트  │ │ 스트레스  │             │

│       │  └─────────────┘ └─────────────────┘               │
│       │  (이미 다른 Goal에서 사용 중인 WHY)                  │
│       │                                                     │
│       │  ┌─────────────────┐                                │
│       │  │  + 직접 입력     │                                │
│       │  └─────────────────┘                                │

── WHY 선택 규칙 ─────────────────────────────────────────────
* AI 추천 WHY: 선택한 카테고리/WHAT 맥락 기반 새 WHY 추천
* 기존 WHY (🔗): 사용자가 다른 Goal에서 이미 생성한 WHY 노드
  - 카드에 연결된 Goal 수 표시 (예: "공부 외 1")
  - 선택 시 해당 WHY 노드에 새 Goal이 연결됨 (Cross-Goal 자동 생성)
  - 기존 WHY가 없으면 이 섹션 숨김
* 복수 선택 가능 (AI 추천 + 기존 WHY 혼합 선택 가능)
* AI 로딩 중: 기존 WHY 먼저 표시, AI 추천 영역 skeleton shimmer
```

**인터랙션:**

| 액션 | 동작 |
| ---- | ---- |
| [+ 새 목표 추가] | 카테고리 선택 영역이 트리에 나타남 |
| 카테고리 클릭 | 카테고리 노드 생성 → 아래에 WHAT 구체화 옵션 (고정 + AI) |
| WHAT 옵션 클릭 | WHAT 노드 생성 → 아래에 WHY 옵션 (AI 추천 + 기존 WHY) |
| WHY 옵션 클릭 | WHY 노드 생성/연결 → 다음 질문 (HOW/중요도/상태) |
| 🔗 기존 WHY 클릭 | 기존 WHY 노드에 새 Goal 연결 (Cross-Goal 자동 생성) |
| + 직접 입력 | 텍스트 입력 필드 표시 → Enter로 노드 생성 |
| 생성된 노드 클릭 | 해당 선택 재변경 |
| 복수 선택 (WHAT) | 각 선택마다 독립 Goal 생성 → 병렬 분기 |
| 복수 선택 (WHY) | 하나의 Goal에 여러 WHY 연결 → Cross-Goal 매핑 |
| 모든 필수 항목 완료 | Goal 카드 노드로 확정 |
| [취소] | 생성 중인 노드들 삭제 |

**Goal 카드 구성:**

| 요소 | 위치 | 설명 |
| ---- | ---- | ---- |
| 카테고리 아이콘 + 제목 | 좌상단 | 💪 러닝 |
| 중요도 라벨 | 우상단 | 🔴 필수 / ⚪ 선택 |
| WHY | 본문 상단 | 복수 가능 (· 구분) |
| HOW | 본문 | 어떻게/얼마나 **(스킵 시 행 숨김)** |
| 전체 진도 | 본문 | 📍 진행률 바 (전체 Phase 기준) |
| 현재 Phase + 코칭 | 본문 | 현재 단계 강조 + 💬 코칭 메시지 |
| Actions | 본문 하단 | 현재 Phase의 체크리스트 (🔗 = 공유 Action) |
| 다음 단계 | Actions 하단 | 전체 공개 (muted 스타일) |
| 🔗 연결된 목표 | 하단 | Cross-Goal 매핑 |
| 버튼들 | 최하단 | ✨ 확장하기, 📝 기록, 📅 일정, ⋯ 더보기 |

**카드 상태:**

| 상태 | 표현 |
| ---- | ---- |
| 펼침 (Default) | 전체 정보 + Actions 표시 |
| 접힘 | 제목 + WHY만 표시 + [펼치기] |
| Hover | 그림자 증가 |

**카드 인터랙션:**

| 클릭 대상 | 동작 |
| ---- | ---- |
| 카드 전체 영역 | S-DETAIL 모달 열기 |
| Actions 체크박스 | 체크 토글만 (S-DETAIL 열리지 않음) |
| [⋯] 더보기 | 드롭다운 메뉴 (삭제/편집 등 빠른 액션) |
| [+ 새 목표 추가] 카드 | **트리 뷰로 전환** 후 인라인 Goal 생성 플로우 진입 (Goal 생성은 항상 트리 뷰에서만) |

**자동 배치 규칙 (Auto-Layout):**

```text
새 카테고리 추가 시:

  [추가 전]                              [추가 후]
  ┌──────────┐                           ┌──────────┐         ┌──────────┐
  │ 💪 운동  │  (중앙 배치)              │ 💪 운동  │         │ 💼 공부  │
  └──────────┘                           └──────────┘         └──────────┘
                                              ◀──── gap: 160px ────▶
```

| 규칙 | 설명 |
| ---- | ---- |
| 자동 재배치 | 카테고리 추가/삭제 시 모든 노드가 수평으로 자동 재배치 |
| 배치 방향 | 기존 노드는 좌측으로 밀리며 새 노드는 우측에 배치 |
| 하위 노드 정렬 | WHAT, WHY, HOW 노드도 상위 노드 아래에 자동 정렬 |
| 최소 간격 | 카테고리 간 160px, WHAT 노드 간 80px |
| 애니메이션 | 300ms ease-in-out (기존 노드 이동 + 새 노드 등장 동시) |
| 화면 초과 시 | 자동 줌아웃 또는 패닝 안내 |

**인터랙션:**

| 액션 | 설명 |
| ---- | ---- |
| 노드 클릭 | Goal 상세 모달 열기 |
| 노드 드래그 | 위치 이동 (수동 배치, 해당 노드 자동 배치에서 제외) |
| 줌/팬 | 마우스 휠 + 드래그 |
| 맞춤 버튼 | 전체 트리가 화면에 맞게 조정 |
| 새 카테고리 추가 | 기존 노드 좌측 밀림 → 새 카테고리 우측 배치 (300ms) |
| 카테고리 삭제 | 빈 공간 자동 채움 → 나머지 노드 재배치 |

### 공유 Action (Cross-Goal)

> 하나의 Action이 여러 Goal의 Phase에 동시 연결되는 경우의 표시 방식

```text
┌─ 💼 취업 준비 ──────────────┐    ┌─ 🎨 오픈소스 기여 ───────────┐
│                              │    │                              │
│  1️⃣ 자기분석 ◀── 지금 여기   │    │  1️⃣ 프로젝트 탐색 (완료)     │
│     ☑ 🔗 Java Spring 공부   │    │  2️⃣ 기여 시작 ◀── 지금 여기   │
│     ☐ 관심 직무 탐색          │    │     ☑ 🔗 Java Spring 공부   │
│                              │    │     ☐ 첫 PR 제출             │
└──────────────────────────────┘    └──────────────────────────────┘

* "Java Spring 공부" 는 하나의 Action (독립 엔티티)
* 취업준비 Phase 1, 오픈소스 Phase 2에 동시 연결
* 한쪽에서 체크 → 양쪽 모두 완료 처리 → 양쪽 진행도 반영
* 🔗 아이콘으로 다른 Goal에도 연결된 Action임을 표시
```

## 8. 상태 정의

### 공통 상태

| 상태 | 표현 |
| ---- | ---- |
| Loading | Skeleton UI (shimmer) |
| Empty | 안내 메시지 + CTA |
| Error | 에러 메시지 + 재시도 버튼 |
| Success | Toast 알림 |

### Goal 카드 상태

| 상태 | 표현 |
| ---- | ---- |
| Default | 기본 카드 스타일 |
| Hover | 그림자 증가 |
| 펼침 | 전체 정보 표시 |
| 접힘 | 제목 + WHY만 |
| Editing | 인라인 편집 모드 |

### Phase 상태

| 상태 | 표현 |
| ---- | ---- |
| Upcoming | opacity 0.6 + muted 스타일, 내용은 보임 |
| In Progress | 강조 테두리 + "지금 여기" 표시 + 코칭 메시지 |
| Completed | 체크 아이콘 + 완료 색상 |

### 성찰 페이지 상태

| 상태 | 표현 |
| ---- | ---- |
| 오늘 미기록 | 퀵 기록 카드 활성 (감정 선택 가능) |
| 오늘 기록 완료 | "✅ 오늘 기록 완료" + 기분 아이콘 + [다시 기록하기] |
| 기록 없음 (Empty) | 안내 메시지 + CTA ("첫 번째 성찰을 남겨보세요") |

### 헤더 📝 아이콘 상태

| 상태 | 표현 |
| ---- | ---- |
| 오늘 미기록 | 📝 아이콘 옆 dot(●) 표시 (Sunrise Gold) |
| 오늘 기록 완료 | dot 없음 (기본 상태) |
| 현재 페이지가 /journal | 아이콘 active 색상 (Sunrise Gold) |

### Action 체크 상태

| 상태 | 표현 |
| ---- | ---- |
| 미완료 | ☐ 빈 체크박스 |
| 완료 | ☑ 체크 + 취소선 |
| 공유 | 🔗 아이콘 (다른 Goal에도 연결) |
| 예정됨 | 📅 아이콘 (캘린더 연동됨) |

### Toast 알림

| 유형 | 색상 | 지속 |
| ---- | ---- | ---- |
| Success | 녹색 | 3초 |
| Info | 파란색 | 4초 |
| Warning | 노란색 | 5초 |
| Error | 빨간색 | 수동 닫기 |
